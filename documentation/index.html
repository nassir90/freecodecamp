Concurrency Multithreading and Asynchronicity

Concurrency simply refers to doing multiple tasks at the same time, that is to say, it is equivalent to the term "multitasking"

As single or double thread capabilties and synchronousness and asynchronousness can be thought of as two binary properties, there are four possible means of accomplishing concurrency

  1. Single threaded, synchronous: Do one task at a time on one thread e.g. morning routine
  2. Single threaded, asynchronous: Start and stop tasks for a designated reason, such as priority. Think of a the plebian and VIP lines boarding an airplane. There is one line, but the plebian line is stopped occasionally to allow the VIPs inside
  3. Multi threaded, synchronous: Do tasks to completion on more than one thread
  4. Multi threaded, asynchronous: Do #2 on multiple threads

CSS Is more Powerful than You Thought

Sibling selectors in CSS enable you to do things that you may have thought required Javascript

Error and Success messages example

Consider a scenario in which you want to display an error message or a success message depending on what goes into an input box. Some slow individuals may posit that one must attach an onChange callback to each input box, but these individuals are myopic and quite frankly, insane.

What one ought to do is wrap the input, the error message and the success message inside a common parent and let CSS guide the way

The :placeholder-shown pseudoclass is used as a surrogate test as to whether or not an input box is empty. If so, use sibling selectors to set the display of both the error and success messages to none

Then consider the conditions under which the error and successs messages are invisible:

  • The error message is invisible when the contents of the input are :not(:invalid)
  • The success message ins invisible when the contents of the input are :invalid

Simply encode this into css and you'll have yourself Javascript-less error and success indicators!!!

Systemd Systemctl and Service

Systemd is the papa process. It's the daemon run with PID=1, and is used as a replacement for SysV which itself was a tool for managing services

Systemctl is the management program provided directly by systemd

Service is a wrapper around service managing programs, and may use systemctl if it is available. Of course, as a wrapper, it has less functionality than systemctl itself

Use systemctl for specificity and service in scripts for cross-compatibility
